# Local V2-16K From-Zero Spec

## Current decision

The old 4090 Phase 1 checkpoint is treated as lost. The project now uses a local 4070-compatible route:

- fixed `memory.n_slots = 16_384`
- `memory.use_lsh = false` for exact retrieval during debugging
- `memory.online_update = false`
- `seq_len = 256` for the first local run
- staged data budgets: `local_v2_1m`, `local_v2_3m`, `local_v2_5m`
- no Phase 2 until Phase 1-local and Phase 1.5B memory objective pass

The purpose is not to recover the old 215K-memory experiment. The purpose is to prove, on local hardware, that a small exact LDMG can be trained with local objectives and audited at every step.

## Data definition

With `seq_len=256`, the named stages are:

| Stage | Train blocks | Approx train tokens |
|---|---:|---:|
| `local_v2_1m` | 4096 | 1,048,576 |
| `local_v2_3m` | 12288 | 3,145,728 |
| `local_v2_5m` | 20480 | 5,242,880 |

Validation and test are generated by `split_counts()` in the existing pipeline. Raw documents are cached under `data/raw_cache/<stage>/...`, so reruns should use `--reuse_raw_cache`.

## Source mix

The domain mix remains hard-coded in `scripts/data/common.py`:

- general: 85%
- reasoning: 10%
- divergent: 5%

Remote source mix is specified in `data/sources/local_v2_sources.yaml`:

General:
- `HuggingFaceFW/fineweb-edu`
- `HuggingFaceTB/smollm-corpus`, subset `fineweb-edu-dedup`
- `HuggingFaceTB/smollm-corpus`, subset `cosmopedia-v2`

Reasoning:
- `nvidia/OpenMathReasoning`, split `cot`, formatted as problem + solution
- `Nan-Do/code-search-net-python`

Divergent:
- local deterministic synthetic divergent documents generated by the pipeline

Reasoning split for OpenMathReasoning must remain `cot`, not `train`.

## Execution order

### 0. Local synthetic memory smoke

This verifies the V2 memory objective implementation without data download or Phase 1 checkpoint.

```bash
python scripts/run_local_v2_16k_from_zero.py --stage synthetic_memory_smoke
```

Pass criteria:
- no NaN/OOM
- `outputs/local_v2_synthetic_memory_smoke/metrics.jsonl` exists
- memory values/query/key grad norms are nonzero

### 1. Data pipeline

For 1M tokens:

```bash
python scripts/run_local_v2_16k_from_zero.py --stage data --mtokens 1 --reuse_raw_cache
python scripts/run_local_v2_16k_from_zero.py --stage audit_data --mtokens 1
```

If there is no internet, add `--offline_mock` for pipeline smoke only. Offline mock must not be treated as real training data.

### 2. Phase 1-local

Start small:

```bash
python scripts/run_local_v2_16k_from_zero.py --stage phase1 --mtokens 1 --phase1_steps 512
```

Then increase only after the first run is stable:

```bash
python scripts/run_local_v2_16k_from_zero.py --stage phase1 --mtokens 1 --phase1_steps 2000
```

Use `configs/local_v2_16k_phase1_4070.yaml` first. If it OOMs, use:

```bash
--phase1_config configs/local_v2_16k_phase1_4070_safe.yaml
```

The safe config disables dense feedback to reduce memory and compute. It is for smoke/stability only.

### 3. Feature extraction

```bash
python scripts/run_local_v2_16k_from_zero.py --stage features --mtokens 1 --feature_samples 1000
```

This extracts span-level features:
- `z_query_l5`
- `z_fused_l5`

### 4. Phase 1.5B memory objective dry run

```bash
python scripts/run_local_v2_16k_from_zero.py --stage memory --mtokens 1 --memory_steps 50
```

Pass criteria:
- reconstruction loss drops at least 5% in 50 steps
- `memory.values_grad_norm > 1e-4`
- `memory.query_proj_grad_norm > 1e-5`
- `memory.key_proj_grad_norm > 1e-5`
- retrieval entropy begins to decrease
- no NaN/OOM

## One-command path

After synthetic smoke has passed, a complete short run is:

```bash
python scripts/run_local_v2_16k_from_zero.py \
  --stage all \
  --mtokens 1 \
  --reuse_raw_cache \
  --phase1_steps 512 \
  --feature_samples 1000 \
  --memory_steps 50
```

Do not use `--stage all` for the first attempt if data download has not been tested.

## Stop rules

Stop and inspect before continuing if any of these happen:

- data audit decision is not PASS
- Phase 1 loss has NaN/Inf
- Phase 1 CUDA OOMs
- checkpoint is not saved
- feature extraction has NaN/Inf or zero norms
- memory objective recon loss does not decrease
- query/key grad norms remain around zero

## What not to do

- Do not use 215K memory locally yet.
- Do not enable `online_update`.
- Do not start Phase 2.
- Do not use LSH during 16K debugging.
- Do not merge `DP_work` into main.
- Do not commit data, raw cache, features, metrics, or checkpoints.
